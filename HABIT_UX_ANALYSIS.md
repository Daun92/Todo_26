# 습관 관리 및 행동 기록 UX 분석 및 개선 계획

> 작성일: 2025-12-15

## 1. 현재 시스템 분석

### 1.1 습관 트래커 (HabitTracker)

#### 현재 기능
```
[홈] → [오늘의 습관 카드] → [아이콘 탭] → [완료 표시]
```

| 항목 | 현황 |
|------|------|
| 습관 항목 | 5개 고정 (걷기, 자세, 코어, 호흡, 명상) |
| 인터랙션 | 탭으로 토글 (ON/OFF) |
| 피드백 | 체크 아이콘, 색상 변화, 프로그레스 바 |
| 데이터 저장 | 일별 HabitLog (habits: Record<string, boolean>) |
| 히스토리 | Heatmap으로 시각화 |

#### 유저 플로우 (As-Is)
```
1. 앱 접속
2. 홈 화면에서 습관 카드 확인
3. 습관 아이콘 탭하여 완료 처리
4. 프로그레스 바로 달성률 확인
5. (끝)
```

#### 한계점
| 문제 | 설명 |
|------|------|
| ❌ 단순 ON/OFF | 강도/품질 기록 불가 (예: 30분 걸었다 vs 10분 걸었다) |
| ❌ 시간 기록 없음 | 언제 수행했는지 추적 불가 |
| ❌ 습관 추가/수정 불가 | 고정된 5개 습관만 사용 |
| ❌ 리마인더 없음 | 알림/푸시 기능 없음 |
| ❌ 맥락 기록 없음 | "오늘 걷기가 힘들었던 이유" 등 기록 불가 |
| ❌ 습관 연결 없음 | 목표와의 연결 관계 없음 |
| ❌ 게이미피케이션 부재 | 보상/레벨 시스템 없음 |

### 1.2 챌린지 시스템 (ChallengeSection)

#### 현재 기능
```
[홈] → [오늘의 챌린지 카드] → [챌린지 선택] → [완료 모달] → [기록 입력] → [저장]
```

| 항목 | 현황 |
|------|------|
| 챌린지 유형 | 일간 4개, 주간 2개, 월간 2개 |
| 완료 시 입력 | 제목, 출처, 핵심 내용, 인사이트, 적용 계획 |
| 상태 | pending, in_progress, completed, skipped |
| 목표 연결 | linkedGoals 필드 존재 (미사용) |

#### 유저 플로우 (As-Is)
```
1. 앱 접속
2. 홈 화면에서 오늘의 챌린지 확인
3. 챌린지 클릭 → 완료 모달 오픈
4. 제목 입력 (필수)
5. 출처, 내용, 인사이트, 적용 계획 입력 (선택)
6. 완료 버튼 클릭
7. 목록에서 완료 표시 확인
```

#### 한계점
| 문제 | 설명 |
|------|------|
| ❌ 진행 중 상태 없음 | 시작했지만 완료 안 한 챌린지 추적 불가 |
| ❌ 건너뛰기 기록 없음 | 왜 건너뛰었는지 기록 불가 |
| ❌ 목표 연결 미작동 | linkedGoals 선택 UI 없음 |
| ❌ AI 인사이트 미연동 | extractChallengeInsights() 미사용 |
| ❌ 템플릿 수정 불가 | 사용자 커스터마이징 없음 |
| ❌ 회고 연결 없음 | 완료 기록이 주간/월간 회고에 자동 반영 안 됨 |

## 2. 유저 경험 페인 포인트

### 2.1 Friction Points (마찰점)

```
🔴 High Friction (높은 마찰)
├── 챌린지 완료 시 입력 필드 과다 (5개 필드)
├── 습관 커스터마이징 불가
└── 진행 상황 추적 어려움

🟡 Medium Friction (중간 마찰)
├── 목표와 행동 간 연결 부재
├── 히스토리 조회 제한적
└── 피드백 부족

🟢 Low Friction (낮은 마찰)
├── 습관 토글은 간편
└── 일간 진행률 표시 명확
```

### 2.2 사용자 시나리오별 문제

#### 시나리오 A: "아침 루틴 수행"
```
현재: 앱 열기 → 각 습관 탭 5번 → 끝
문제:
- 순서나 시간 기록 없음
- "오늘 명상 대신 호흡에 집중" 같은 맥락 없음
- 아침/저녁 구분 없음
```

#### 시나리오 B: "기사 리뷰 챌린지 완료"
```
현재: 챌린지 클릭 → 5개 필드 입력 → 완료
문제:
- 필수 입력이 제목뿐이라 나머지 스킵 유도
- 인사이트 입력이 번거로움 (개별 추가 필요)
- 목표와 연결 수동으로 불가
```

#### 시나리오 C: "주간 회고 시"
```
현재: 인사이트 페이지 → AI 회고 생성
문제:
- 습관 데이터는 완료 여부만
- 챌린지 기록은 있지만 AI 분석 미적용
- 어떤 행동이 어떤 성장에 기여했는지 추적 어려움
```

## 3. 개선 방향

### 3.1 습관 트래커 개선

#### Phase 1: Quick Wins (즉시 적용 가능)
| 개선 항목 | 설명 | 우선순위 |
|-----------|------|----------|
| 습관 길게 눌러 메모 | 롱프레스 시 간단 메모 입력 | 🔴 높음 |
| 에너지/무드 기록 | 습관 완료 후 1-5 스케일 | 🔴 높음 |
| 완료 시간 자동 저장 | completedAt 타임스탬프 | 🟡 중간 |
| 완료 애니메이션 강화 | Lottie 또는 CSS 애니메이션 | 🟢 낮음 |

#### Phase 2: Medium-term (중기)
| 개선 항목 | 설명 |
|-----------|------|
| 습관 추가/편집 UI | 사용자 정의 습관 생성 |
| 습관-목표 연결 | 각 습관이 어느 목표에 기여하는지 |
| 루틴 그룹핑 | 아침/저녁/업무 중 습관 분류 |
| 습관 스트릭 개별 표시 | 각 습관별 연속 기록 |

#### Phase 3: Long-term (장기)
| 개선 항목 | 설명 |
|-----------|------|
| AI 습관 코칭 | "오늘 명상 대신 호흡에 집중해보세요" |
| 스마트 리마인더 | 시간/장소 기반 알림 |
| 습관 상관관계 분석 | "코어 운동 후 자세가 좋아지는 경향" |

### 3.2 챌린지 시스템 개선

#### Phase 1: Quick Wins
| 개선 항목 | 설명 | 우선순위 |
|-----------|------|----------|
| 간편 완료 모드 | 제목만 입력하고 빠르게 완료 | 🔴 높음 |
| 인사이트 텍스트 영역 | 개별 추가 대신 여러 줄 입력 | 🔴 높음 |
| 목표 선택 드롭다운 | linkedGoals 연결 UI | 🟡 중간 |
| "건너뛰기" 버튼 | 사유와 함께 스킵 기록 | 🟡 중간 |

#### Phase 2: Medium-term
| 개선 항목 | 설명 |
|-----------|------|
| AI 인사이트 추출 | 완료 시 자동으로 핵심 포인트 제안 |
| 진행 중 상태 | 시작했지만 완료 안 한 챌린지 표시 |
| 챌린지 템플릿 편집 | 사용자 정의 챌린지 생성 |
| 리마인더 설정 | 챌린지별 알림 시간 |

#### Phase 3: Long-term
| 개선 항목 | 설명 |
|-----------|------|
| 챌린지 난이도 조정 | AI가 수준에 맞는 챌린지 제안 |
| 소셜 챌린지 | 친구와 함께 도전 |
| 챌린지 스트릭 보상 | 연속 완료 시 배지/레벨업 |

## 4. 데이터 모델 개선안

### 4.1 HabitLog 확장
```typescript
interface HabitLog {
  id: string;
  date: string;
  habits: Record<string, HabitEntry>;  // boolean → HabitEntry
  energy?: number;      // 1-5 (오늘의 에너지)
  mood?: number;        // 1-5 (오늘의 기분)
  note?: string;        // 오늘의 간단 메모
}

interface HabitEntry {
  completed: boolean;
  completedAt?: string;  // 완료 시간
  duration?: number;     // 수행 시간 (분)
  quality?: number;      // 1-5 품질/강도
  note?: string;         // 개별 메모
}
```

### 4.2 ChallengeLog 확장
```typescript
interface ChallengeLog {
  id: string;
  templateId: string;
  date: string;
  status: 'pending' | 'in_progress' | 'completed' | 'skipped';
  // 기존 필드들...

  // 추가 필드
  startedAt?: string;      // 시작 시간
  skipReason?: string;     // 건너뛰기 사유
  difficulty?: number;     // 체감 난이도 1-5
  satisfaction?: number;   // 만족도 1-5
  aiInsights?: {           // AI 분석 결과
    keyPoints: string[];
    applicableActions: string[];
    linkedConcepts: string[];
  };
}
```

## 8. 결론

현재 시스템은 **MVP 수준**으로, 기본적인 기록은 가능하나:

1. **깊이 부족**: 단순 ON/OFF로 행동의 질적 측면 미반영
2. **연결성 부재**: 습관/챌린지와 목표 간 관계 추적 불가
3. **피드백 부족**: 사용자에게 의미 있는 인사이트 제공 미흡
4. **유연성 부족**: 사용자 커스터마이징 불가

**권장 접근법**:
- Phase 1에서 **데이터 모델 확장**과 **UI 마이크로 개선**에 집중
- Phase 2에서 **AI 연동**으로 자동 인사이트 추출
- Phase 3에서 **분석/코칭** 기능으로 차별화

이를 통해 "기록 앱"에서 "성장 파트너"로 진화 가능.
